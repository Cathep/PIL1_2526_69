1. Présentation du projet

IFRI_MentorLink est une application web destinée à mettre en relation les étudiants de l'IFRI qui souhaitent bénéficier ou offrir du mentorat académique et professionnel. La solution permet aux utilisateurs de créer un profil détaillé, de publier des offres ou demandes de mentorat, de recevoir des suggestions de matching et de communiquer via une messagerie intégrée.

Le projet répond à un besoin concret : faciliter l'entraide entre étudiants en tenant compte des compétences maîtrisées, des lacunes, des filières, des niveaux d'étude et des disponibilités.

4. Conception technique

Architecture générale

L'application adopte une architecture client-serveur. Le navigateur envoie des requêtes HTTP au serveur Django, qui traite la logique métier, interagit avec la base de données MySQL et renvoie des pages HTML. La messagerie utilise Django Channels pour gérer les échanges en temps réel.

5. Base de données

La base de données relationnelle contient les entités principales suivantes :

  • users : informations de compte et profil utilisateur.
  • fields_of_study : filières des étudiants.
  • skills : compétences ou matières.
  • user_skills : points forts et lacunes des utilisateurs.
  • user_availabilities : disponibilités horaires habituelles.
  • mentoring_requests : offres et demandes de mentorat.
  • match_suggestions : résultats de matching.
  • conversations et messages : messagerie intégrée.

Le schéma SQL final est disponible dans le fichier sql/schema_final.sql.

6. Algorithme de matching

L'algorithme compare l'utilisateur connecté avec les autres utilisateurs actifs. Il évalue :

  • la correspondance entre les lacunes d'un mentoré et les compétences d'un mentor ;
  • le chevauchement des disponibilités horaires ;
  • la proximité de filière ;
  • la proximité de niveau d'étude.

Le score global est calculé avec une pondération : 45 % compétences, 30 % horaires, 15 % filière et 10 % niveau. Les résultats sont ensuite triés par score décroissant.

7. Sécurité et confidentialité

  • Les mots de passe sont hashés par le système d'authentification Django.
  • Les emails et numéros de téléphone sont uniques.
  • Les pages principales sont protégées par authentification.
  • Les conversations ne sont accessibles qu'aux utilisateurs participants.
  • Les données personnelles affichées sont limitées aux informations utiles au mentorat.

8. Instructions de déploiement

Prérequis

  • Python 3.12 ou version compatible.
  • MySQL ou MariaDB.
  • Git.

Installation

git clone https://github.com/Cathep/PIL1_2526_69.git
cd PIL1_2526_69
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Configuration de la base de données

CREATE DATABASE mentorlink CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'mentorlink_user'@'localhost' IDENTIFIED BY 'Mentorlink2025';
GRANT ALL PRIVILEGES ON mentorlink.* TO 'mentorlink_user'@'localhost';
FLUSH PRIVILEGES;

Démarrage

python manage.py migrate
python manage.py createsuperuser
python manage.py runserver

L'application est ensuite disponible à l'adresse http://127.0.0.1:8000/.

9. Manuel d'utilisation

Créer un compte

Accéder à la page d'inscription, renseigner le nom, le prénom, l'email, le téléphone et le mot de passe. Après inscription, compléter le profil avec la filière, le niveau, les compétences, les lacunes et les disponibilités.

Publier une offre ou une demande

Depuis le tableau de bord, accéder à la page des publications puis créer une offre ou une demande en choisissant la compétence concernée, le format de séance et les horaires souhaités.

Lancer le matching

Utiliser l'action de matching pour générer automatiquement des suggestions de mentors ou mentorés. Les résultats affichent le score global, les compétences communes et les disponibilités compatibles.

Discuter avec un utilisateur

Après acceptation d'un matching ou depuis un profil utilisateur, ouvrir une conversation et envoyer des messages. L'historique reste disponible dans la liste des conversations.

10. Limites et perspectives

  • La couche de temps réel utilise une configuration simple adaptée au développement.
  • Le système de notification peut être amélioré avec un compteur global de messages non lus.
  • Le matching pourrait intégrer des critères supplémentaires, comme les centres d'intérêt académiques.
  • Le déploiement en production nécessiterait une configuration plus sécurisée des variables sensibles.
